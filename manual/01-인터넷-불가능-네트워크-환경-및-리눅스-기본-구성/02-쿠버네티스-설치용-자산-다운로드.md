# 01-03. 쿠버네티스 설치용 자산 다운로드

## 01-03.1. 환경변수 파일 준비
```bash
cd <프로젝트-루트>
pwd
make 01-02-env-file-create
vi .env
source ops/01-airgap-linux-environment/scripts/project-env-setup.sh
```

설명: `.env`는 `<프로젝트-루트>/.env`에 생성된다. 키 파일 경로를 입력한 뒤 `project-env-setup.sh`로 Terraform output 동기화와 현재 셸 로드를 함께 수행한다.

## 01-03.2. 공식문서 링크 확인
```text
https://kubernetes.io/releases/
https://kubernetes.io/docs/setup/production-environment/tools/kubeadm/install-kubeadm/
https://kubernetes.io/docs/setup/production-environment/container-runtimes/
https://kubernetes.io/docs/reference/setup-tools/kubeadm/kubeadm-init/
https://kubernetes.io/docs/reference/setup-tools/kubeadm/generated/kubeadm_config/kubeadm_config_images_list/
https://docs.tigera.io/calico/latest/getting-started/kubernetes/self-managed-onprem/onpremises
```

설명: Kubernetes `v1.35.4` 설치와 오프라인 자산 산정 기준이 되는 공식문서 링크를 확인한다.

## 01-03.3. 버전 고정값 확인
```text
Kubernetes patch version: 1.35.4
kubeadm/kubelet/kubectl rpm repo: pkgs.k8s.io core:/stable:/v1.35/rpm
container runtime package source: Amazon Linux 2023 dnf repo
asset format: rpm
Calico manifest version: v3.31.4
```

설명: 실제 다운로드와 검증에 사용한 고정 버전 기준이다.

## 01-03.4. 작업 원본 자산 경로 확인
```bash
find assets/offline-assets -maxdepth 3 -type d | sort
```

설명: 로컬 PC에서 다운로드한 패키지, 이미지, manifest를 저장할 작업 원본 경로를 확인한다.

## 01-03.5. 서버 반입 번들 경로 확인
```bash
find delivery/offline-assets -maxdepth 2 -type d | sort
ls -lh delivery/offline-assets.tar.gz
```

설명: 서버로 실제 반입할 디렉터리 번들과 압축본 경로를 확인한다.

## 01-03.6. 서버 설치 기준 경로 확인
```text
/opt/offline-assets
```

설명: VM 또는 폐쇄망 서버에서 압축을 해제하고 설치를 수행할 기준 경로다.

## 01-03.7. 다운로드 대상 분류 확인
```text
assets/offline-assets/kubernetes/packages/container-runtime/
assets/offline-assets/kubernetes/packages/kubernetes/
assets/offline-assets/kubernetes/images/kube-system/
assets/offline-assets/kubernetes/manifests/
assets/offline-assets/common/checksums/
```

설명: containerd rpm, Kubernetes rpm, kube-system 이미지, CNI manifest, checksum 저장 위치를 구분한다.

## 01-03.8. 다운로드 방식 기준 확인
```text
로컬 PC: Docker를 사용해 격리된 Amazon Linux 2023 컨테이너에서 패키지를 다운로드
클러스터 노드: containerd를 Kubernetes container runtime으로 사용
```

설명: Docker는 로컬 PC에서 오프라인 자산을 깨끗하게 수집하기 위한 보조 도구다. 실제 Kubernetes 노드는 Amazon Linux 2023을 사용하고, 패키지 형식은 `rpm/dnf`, container runtime은 `containerd`를 기준으로 설치한다.

## 01-03.9. 쿠버네티스 설치용 자산 다운로드 실행
```bash
make 01-03-offline-assets-run
```

설명: 작업 원본 경로 확인부터 Kubernetes 패키지, runtime 패키지, Calico manifest, kube-system 이미지 다운로드와 반입 번들 생성/검증까지 한 번에 수행한다.

## 01-03.10. 다운로드 결과 검증
```bash
make 01-03-offline-assets-verify
```

설명: `.rpm`, manifest, image tar, checksum 파일과 반입 압축본이 실제로 생성되었는지 확인한다.

## 01-03.11. 주요 산출물 확인
```bash
find assets/offline-assets/kubernetes/packages/kubernetes -maxdepth 1 -name '*.rpm' | sort
find assets/offline-assets/kubernetes/packages/container-runtime -maxdepth 1 -name '*.rpm' | sort
find assets/offline-assets/kubernetes/images/kube-system -maxdepth 1 -name '*.tar' | sort
find assets/offline-assets/kubernetes/manifests -maxdepth 1 -name '*.yaml' | sort
```

설명: 실제 다운로드된 `rpm` 패키지, 이미지, manifest 파일 목록을 확인한다.

## 01-03.12. 반입 번들 생성 기준 확인
```bash
make 01-03-offline-assets-clear
make 01-03-offline-assets-run
```

설명: 오프라인 자산 준비 단계를 처음부터 다시 재현할 때 사용하는 기준 명령이다.

## 01-03.13. 세부 실행이 필요한 경우
```bash
cd ops/01-airgap-linux-environment
make help
make step-01-03-01-assets-tree-check
make step-01-03-02-k8s-assets-download
make step-01-03-03-k8s-assets-verify
make step-01-03-04-bundle-create
make step-01-03-05-bundle-verify
make 01-03-offline-assets-clear
```

설명: 다운로드 단계와 번들 단계를 나누어 점검하거나 특정 단계만 다시 실행할 때는 `ops/01-airgap-linux-environment/Makefile` 기준으로 실행한다.
