# 전체 실행 순서

## 전체 실행 명령 순서
```bash
make 00-01-project-root-check
make 01-01-terraform-run
make 01-01-terraform-verify
make 01-02-env-file-create
source ops/01-airgap-linux-environment/scripts/project-env-setup.sh
make 01-02-env-vars-verify
make 01-03-offline-assets-run
make 01-03-offline-assets-verify
```

설명: 위 명령까지 끝나면 로컬 준비 단계가 완료된다. 이후 `02-01`, `03-01`은 장별 상세 문서 기준으로 원격 노드에서 진행한다.

## 00-01. 프로젝트 루트 확인
```bash
make 00-01-project-root-check
```

설명: 전체 명령은 `airgap-k8s` 프로젝트 루트에서 실행한다.
검증 기준: `pwd` 출력 경로가 현재 작업한 프로젝트 루트이고, `ls` 출력에 `manual/`, `ops/`, `assets/`, `delivery/`가 보여야 한다.

## 01-01. Terraform 실습 환경 생성
```bash
make 01-01-terraform-run
```

설명: `init -> validate -> plan -> apply`를 한 번에 실행한다.
검증:
```bash
make 01-01-terraform-verify
```

검증 기준: `bastion_public_ip`, `master_private_ip`, `worker1_private_ip`가 출력되어야 한다.

## 01-02. 환경변수 준비 및 로드
상세: `manual/01-인터넷-불가능-네트워크-환경-및-리눅스-기본-구성/README.md`

```bash
make 01-02-env-file-create
vi .env
source ops/01-airgap-linux-environment/scripts/project-env-setup.sh
```

설명: 최초 1회 `.env`를 만든 뒤 SSH 키 경로를 입력하고, Terraform output 기반 값 갱신과 현재 셸 로드를 한 번에 수행한다.
검증:
```bash
make 01-02-env-vars-verify
```

검증 기준: `AIRGAP_`로 시작하는 환경변수가 출력되고, `AIRGAP_SSH_USER`, `AIRGAP_SSH_KEY_PATH`, `AIRGAP_BASTION_PUBLIC_IP`, `AIRGAP_MASTER_PRIVATE_IP`, `AIRGAP_WORKER1_PRIVATE_IP` 값이 비어 있지 않아야 한다.

## 01-03. 오프라인 자산 준비
상세: `manual/01-인터넷-불가능-네트워크-환경-및-리눅스-기본-구성/02-쿠버네티스-설치용-자산-다운로드.md`

```bash
make 01-03-offline-assets-run
```

설명: 작업 원본 경로 확인, Kubernetes 오프라인 자산 다운로드, 다운로드 검증, 서버 반입용 번들 생성, 번들 검증을 한 번에 수행한다.
검증:
```bash
make 01-03-offline-assets-verify
```

검증 기준: `delivery/offline-assets.tar.gz` 파일이 생성되고, 아카이브 안에 `offline-assets/`로 시작하는 항목이 출력되어야 한다.

세부 실행:
```bash
cd ops/01-airgap-linux-environment
make help
```

설명: `01-03` 세부 step은 `ops/01-airgap-linux-environment/Makefile` 기준으로 실행한다.

## 02-01. 사용자 및 네트워크 설정
상세: `manual/02-사용자-및-네트워크-설정/README.md`

```bash
source ops/01-airgap-linux-environment/scripts/load-project-env.sh
```

설명: `devops`, hostname, `/etc/hosts` 설정은 `ops/02-user-network/scripts/` 아래 스크립트를 각 노드에 업로드한 뒤 실행한다.
검증 기준: master와 worker 모두 `verify-node-user-network.sh` 결과가 `[RESULT] SUCCESS`여야 한다.

세부 실행:
```bash
cd ops/02-user-network
make help
```

## 03-01. 쿠버네티스 사전점검과 설치
상세: `manual/03-쿠버네티스-클러스터-구성/README.md`

```bash
source ops/01-airgap-linux-environment/scripts/load-project-env.sh
```

설명: bastion 접속, 자산 반입, `/opt/offline-assets` 배치, kubeadm 공식문서 기준 사전점검은 이 장에서 수행한다.
검증 기준: master와 worker 모두 `kubeadm-preflight-check.sh` 결과가 `failures: 0`이어야 한다.

세부 실행:
```bash
cd ops/03-kubernetes-cluster
make help
```
