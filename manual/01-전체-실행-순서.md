# 전체 실행 순서

## 전체 실행 명령 순서
```bash
make 00-01-project-root-check
make 01-01-terraform-run
make 01-01-terraform-verify
make 01-02-env-file-create
cp manual/00-제출-매뉴얼-개요/03-사전-조사-정보-수집-템플릿.example.yaml \
  manual/00-제출-매뉴얼-개요/03-사전-조사-정보-수집-템플릿.yaml
./ops/03-kubernetes-cluster/common/scripts/render-env-from-survey-yaml.sh
source ops/01-airgap-linux-environment/scripts/project-env-setup.sh
make 01-02-env-vars-verify
make 01-03-offline-assets-run
make 01-03-offline-assets-verify
make ops-runtime-bundle-run
make ops-runtime-bundle-verify
make all-verify
```

설명: 위 명령까지 끝나면 로컬 준비 단계가 완료된다. 이후 `02-01`, `03-01`은 장별 상세 문서 기준으로 원격 노드에서 진행한다. bastion에서 `kubectl`, `helm`, `ansible-playbook`를 실행할 계획이면 `ops-runtime-bundle-*`까지 같이 준비한다.

```bash
make all-verify
```

설명: 현재 시점에서 Terraform, `.env`, 오프라인 자산, bastion 실행 원본, 실제 적용 마커 상태를 한 번에 검증하고 마지막에 현재 단계와 남은 실제 작업 단계를 요약한다.

## 00-01. 프로젝트 루트 확인
```bash
make 00-01-project-root-check
```

설명: 전체 명령은 `airgap-k8s` 프로젝트 루트에서 실행한다.
검증 기준: `pwd` 출력 경로가 현재 작업한 프로젝트 루트이고, `ls` 출력에 `manual/`, `ops/`, `assets/`, `delivery/`가 보여야 한다.

## 01-01. Terraform 실습 환경 생성
```bash
make 01-01-terraform-run
```

설명: `init -> validate -> plan -> apply`를 한 번에 실행한다.
검증:
```bash
make 01-01-terraform-verify
```

검증 기준: `bastion_public_ip`, `master_private_ip`, `worker1_private_ip`가 출력되어야 한다.

## 01-02. 환경변수 준비 및 로드
상세: `manual/01-인터넷-불가능-네트워크-환경-및-리눅스-기본-구성/README.md`

```bash
make 01-02-env-file-create
cp manual/00-제출-매뉴얼-개요/03-사전-조사-정보-수집-템플릿.example.yaml \
  manual/00-제출-매뉴얼-개요/03-사전-조사-정보-수집-템플릿.yaml
vi manual/00-제출-매뉴얼-개요/03-사전-조사-정보-수집-템플릿.yaml
./ops/03-kubernetes-cluster/common/scripts/render-env-from-survey-yaml.sh
source ops/01-airgap-linux-environment/scripts/project-env-setup.sh
```

설명: 최초 1회 조사 YAML을 복사해 실제 값을 입력하고, YAML 기준으로 `.env`를 렌더링한 뒤 Terraform output 기반 값 갱신과 현재 셸 로드를 수행한다.
검증:
```bash
make 01-02-env-vars-verify
```

검증 기준: `AIRGAP_`로 시작하는 환경변수가 출력되고, `AIRGAP_SSH_USER`, `AIRGAP_SSH_KEY_PATH`, `AIRGAP_BASTION_PUBLIC_IP`, `AIRGAP_MASTER_PRIVATE_IP`, `AIRGAP_WORKER1_PRIVATE_IP` 값이 비어 있지 않아야 한다.

## 01-03. 오프라인 자산 준비
상세: `manual/01-인터넷-불가능-네트워크-환경-및-리눅스-기본-구성/02-쿠버네티스-설치용-자산-다운로드.md`

```bash
make 01-03-offline-assets-run
```

설명: 작업 원본 경로 확인, Kubernetes 오프라인 자산 다운로드, 다운로드 검증, 서버 반입용 번들 생성, 번들 검증을 한 번에 수행한다.
검증:
```bash
make 01-03-offline-assets-verify
```

검증 기준: `delivery/offline-assets.tar.gz` 파일이 생성되고, 아카이브 안에 `offline-assets/`로 시작하는 항목이 출력되어야 한다.

세부 실행:
```bash
cd ops/01-airgap-linux-environment
make help
```

설명: `01-03` 세부 step은 `ops/01-airgap-linux-environment/Makefile` 기준으로 실행한다.

## 공통. bastion 실행 원본 번들 준비
```bash
make ops-runtime-bundle-run
```

설명: 작성 원본 `ops/common`, `ops/02`, `ops/03`, `ops/04`, `ops/05`를 `delivery/ops-runtime.tar.gz`로 묶는다.
검증:
```bash
make ops-runtime-bundle-verify
```

검증 기준: `delivery/ops-runtime.tar.gz` 파일이 생성되고, 아카이브 안에 `airgap-k8s-ops/ops/`로 시작하는 항목이 출력되어야 한다.

## 02-01. 사용자 및 네트워크 설정
상세: `manual/02-사용자-및-네트워크-설정/README.md`

```bash
source ops/01-airgap-linux-environment/scripts/load-project-env.sh
```

```bash
make 02-01-user-network-run
make 02-01-user-network-verify
```

설명: `02-01-user-network-run`은 원격 적용 후 같은 단계의 검증 스크립트를 즉시 실행하고, 성공하면 상태 마커를 기록한다.
검증 기준: `02-01-user-network-verify` 결과가 `[RESULT] SUCCESS`여야 한다.

세부 실행:
```bash
cd ops/02-user-network
make help
```

## 03-01. 쿠버네티스 사전점검과 설치
상세: `manual/03-쿠버네티스-클러스터-구성/README.md`

```bash
source ops/01-airgap-linux-environment/scripts/load-project-env.sh
make 03-01-preflight-run
make 03-01-preflight-verify
```

설명: bastion 접속, 자산 반입, `/opt/offline-assets` 배치, kubeadm 공식문서 기준 사전점검은 이 장에서 수행한다.
검증 기준: `03-01-preflight-verify` 결과가 `[RESULT] SUCCESS`여야 한다.

세부 실행:
```bash
cd ops/03-kubernetes-cluster
make help
```

설명: 공통 사전점검은 `ops/03-kubernetes-cluster/common/`, 수동 설치는 `manual-kubeadm/`, 자동화는 `ansible-kubeadm/`을 기준으로 진행한다. 수동과 Ansible은 `01`~`06` 같은 단계 번호를 공유한다.
